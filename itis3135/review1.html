A Checklist for Each Page (from the top)

  1. Your submission leads right to the page to be reviewed
    • The home page loads directly and the navigation links to all client pages (Home, About, Events, Resources, Join).
  2. There are no spaces or upper-cases in the file/folder names
    • Folder and file names appear lowercase with no spaces (for example, mtc_client/index.html).
  3. Design
    1. Page has sufficient contrast / font sizing
      • Text is generally readable. It would still be worth double-checking contrast on all text for accessibility.
    2. Page has site colors and fonts using the standard CSS file
      • Colors and typography are consistent across the site and appear to come from a shared stylesheet.
    3. CRAP
      1. Contrast – overall good, with a few areas that could be refined.
      2. Repetition – navigation and heading styles are consistent on every page.
      3. Alignment – sections line up cleanly and content is easy to scan.
      4. Proximity – related items (events, resources, join information) are grouped logically.
  4. Page has within it:
    1. Header
    2. Main
    3. Footer
    4. (optional) nav for main menu between header and main
    • Each page has a clear top area, navigation, main content, and footer.
    • It would strengthen the structure to explicitly use semantic <header>, <main>, and <footer> tags if those are not already in place.
  5. Header has site/brand header with text in an h1
    1. If this is a branded page it should be the brand/company.
    2. DO NOT include the page name in the header.
    3. Understand the difference between head, header, and heading.
    • “Muslim Technology Collaborative” is clearly shown as the brand on each page.
    • It would be good to ensure this brand name is marked up as the single h1, and page titles like “About Us” or “Events” are h2 elements inside <main>, not repeated in the header.
  6. Main starts with name of page as an h2 and does not include name of site/brand
    • Each page has a clear title (About Us, Events, Resources, Join).
    • Just confirm those titles are coded as h2 in <main> for full compliance with the course pattern.
  7. Page has brand tagline somewhere (consistent on every page)
    • The brand name is consistent, but there is not yet a short tagline.
    • Suggestion: add something like “Connecting Muslim students in tech at UNC Charlotte” under the logo on all pages for a stronger brand identity.
  8. Page has footer with
    1. Menu for user’s pages
    2. HTML validation button/link and validates
    3. CSS validation button/link and validates
    • The footer currently includes copyright information.
    • Adding a small navigation menu in the footer would make it easier to move between pages from the bottom of the site.
    • At this point, I did not see HTML/CSS validation buttons. These still need to be added and connected to the correct URLs once the pages validate.
  9. Page has specific requirements for this assignment
    • The client site feels appropriate for a real organization with Home, About, Events, Resources, and Join pages.
    • The Resources page is especially strong and gives visitors real value.
    • “Show Leadership” and “Hello World” read as placeholders; these should be replaced with actual content or removed before the final version.
    • The Join section would be stronger as a proper form with labels, inputs, and clear confirmation behavior for the user.
