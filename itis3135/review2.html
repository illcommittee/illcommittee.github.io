Johnson, Gabriel

A Checklist for Each Page (from the top)

  1. Your submission leads right to the page to be reviewed
    • The project link goes to the Bible Study home page, which then links to About, Course Summary, Schedule, and Contact.
  2. There are no spaces or upper-cases in the file/folder names
    • Project files such as about.html, course.html, schedule.html, and contact.html are all lowercase with no spaces inside the project folder.
  3. Design
    1. Page has sufficient contrast / font sizing
      • Body text is generally readable and headings stand out. It would still be worth running a contrast checker to be sure everything meets accessibility guidelines.
    2. Page has site colors and fonts using the standard CSS file
      • The Bible Study section has a consistent color theme and typography across all four main pages.
    3. CRAP
      1. Contrast - headings and navigation are visually prominent, which helps orientation.
      2. Repetition - the Bible Study navigation and page layout repeat from page to page, which is good.
      3. Alignment - content blocks are aligned in a simple, centered layout that reads cleanly.
      4. Proximity - related information (modules, readings, contact info) is grouped in logical sections.
  4. Page has within it:
    1. Header
    2. Main
    3. Footer
    4. (optional) nav for main menu between header and main
    • The Bible Study pages have a clear title, navigation bar, main content, and footer.
    • It would be helpful to make sure semantic tags like <header>, <main>, and <footer> are used consistently in the HTML.
  5. Header has site/brand header with text in an h1
    1. If this is a branded page it should be the brand/company.
    2. DO NOT include the page name in the header.
    3. Understand the difference between head, header, and heading.
    • For the client portion, “BibleStudy” works well as a simple brand heading with its own navigation beneath it.
    • Right now, the very top of the page still shows “Gabriel Johnson’s Gallant Jaguar || ITIS 3135” and a long course navigation. For a client site, it would be better if the client brand were the primary header and the student/course header were either removed, moved to a separate course page, or made much smaller so it does not feel like the client’s site is sitting inside the student’s personal site.
  6. Main starts with name of page as an h2 and does not include name of site/brand
    • The Bible Study pages use headings like “Welcome to the Bible Study Program”, “About This Project”, “Course Summary”, and “Study Schedule & Checklist”. These clearly describe each page.
    • It would be good to confirm these headings are implemented as h2 elements within <main>, with the site name reserved for the h1.
  7. Page has brand tagline somewhere (consistent on every page)
    • The copy explains the purpose of the course (“guided reading of the New Testament”, “structured, gentle approach”).
    • Adding a short, repeatable tagline near the Bible Study logo (for example, “Guided New Testament reading for groups and individuals”) would make the branding stronger and more consistent across pages.
  8. Page has footer with
    1. Menu for user’s pages
    2. HTML validation button/link and validates
    3. CSS validation button/link and validates
    • There is a simple footer on the pages, but it does not yet include a footer navigation menu.
    • I did not see HTML or CSS validation links/buttons. Those still need to be added and connected to the correct validator URLs once the pages validate.
  9. Page has specific requirements for this assignment
    • The Bible Study concept is clear: a guided, trackable course study of the New Testament with modules and a schedule.
    • The Course Summary page outlines three modules (Gospels, Acts & Early Church, Letters and Revelation) and gives a clear idea of what the course covers.
    • The Schedule page describes a checklist of readings and mentions that progress is saved locally in the browser, which fits well with the “trackable” requirement.
    • The About page includes basic client information for Luke Travers but could be expanded a bit more with goals, audience, and any constraints the client has for design/functionality.
    • The Contact page gives an email and a simple slideshow area; adding short labels or captions for the images would make that section feel more intentional.

Additional Notes (Stop / Start / Continue)

Stop: relying on the main ITIS 3135 header and long personal navigation on the client pages. It makes the Bible Study site feel secondary to your course site.

Start: letting the Bible Study branding stand alone at the top of the client pages, adding a concise tagline, and building out the Schedule checklist and Contact slideshow with a bit more detail and labeling.

Continue: keeping the content focused and readable, using clear headings, and tying each page directly to the client’s goal of a structured New Testament study.
